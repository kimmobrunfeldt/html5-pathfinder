Dijkstra

Distance:

Shortest path is calculated normally with Dijkstra's algorithm. As you can see, the graph of connected nodes is quite large even in a small map.

Dijkstra's algorithm is not good in maps that contain large empty fields.

Preprocessed Dijkstra

Distance:

Map is preprocessed to reduce the amount of work to find a path. The algorithm doesn't guarantee the shortest path.

The corners are found with my "own" algorithm. After the corners are found, path can be calculated using only the corner nodes. Corner nodes are connected if there is clear space between them. There musnt't be any walls in the rectangle that can be drawn between the corner nodes. Start and end points are added to corner graph also.

When the actual path is drawn, the orange path blocks are filled between the corners. They are not really in the graph.
